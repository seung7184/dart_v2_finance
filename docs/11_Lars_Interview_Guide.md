## 0. 목적

이 인터뷰의 목적은 **숫자 신뢰 검증**이다.

"Lars가 safe-to-spend 숫자를 믿을 조건이 무엇인가"를 파악하는 것이 핵심.
기능 발견이나 수요 조사가 아니다.

인터뷰 후 결정할 것:
- safe-to-spend 정책을 더 보수적으로 할지 판단
- onboarding에서 꼭 필요한 입력만 남기기

---

## 1. 타깃 인터뷰 대상

아래 조건을 **3개 이상** 만족하는 사람:

- [ ] 네덜란드 거주 직장인 (월급쟁이)
- [ ] ING, Rabobank, ABN 중 하나 이상 사용
- [ ] Trading 212, DeGiro, DEGIRO 등 브로커 계좌 보유
- [ ] ETF 또는 인덱스 펀드로 월간 투자 습관 있음
- [ ] 현재 돈 관리에 불만이 있거나 Excel로 직접 정리하고 있음
- [ ] CSV export를 써본 적 있거나 거부감 없음

---

## 2. 인터뷰 세팅

**형식**: 30분, 비대면 (Google Meet / Zoom)
**보상**: 없음 (초기 beta 접근권 약속 가능)
**녹화**: 동의 받을 것, 없으면 노트만
**언어**: 영어 또는 한국어

**인터뷰어 주의사항**:
- 제품을 설명하지 마라 — 먼저 들어라
- "왜요?" 최소 3번은 파고들어라
- 숫자를 믿는다/안 믿는다는 감정적 신호를 잡아라
- "지금 앱이 뭔가요"보다 "지금 뭘 쓰고 있고 어디서 막히는지"에 집중

## 3. 인터뷰 스크립트

### [도입] 2분

> "오늘 시간 내줘서 고마워요. 저는 개인 재무 앱을 만들고 있는데, 구체적인 아이디어 설명보다는 지금 어떻게 돈 관리하는지 듣고 싶어서요. 정답이 없는 질문들이고, 솔직하게 말해줄수록 도움이 돼요."

---

### [파트 1] 현재 상태 파악 — 5분

**Q1. 지금 돈 관리를 어떻게 하고 있어요?**

듣고 싶은 것:
- 어떤 도구를 쓰는지 (앱, Excel, 머릿속)
- 어떤 계좌가 있는지
- 투자 계좌는 어떻게 추적하는지

→ 도구를 쓴다면: "그 앱에서 제일 짜증나는 게 뭐예요?"
→ Excel 쓴다면: "거기서 제일 유지하기 힘든 부분이 뭐예요?"
→ 아무것도 안 한다면: "그래서 지금 돈 상태 파악을 어떻게 해요?"

---

**Q2. 투자 이체가 지출처럼 보인 적 있어요?**

> 예를 들어 ING에서 Trading 212로 €500 보냈는데 가계부 앱이 그걸 "지출 €500"으로 잡은 적 있나요?

듣고 싶은 것:
- 이게 실제로 문제인지, 얼마나 짜증나는지
- 어떻게 처리하는지 (무시? 수동 수정? 앱을 안 씀?)

→ 짜증났다면: "그래서 그 앱 계속 쓰고 있어요, 아니면 포기했어요?"
→ 문제 아니라면: "그럼 투자 이체를 어떻게 구분해요?"

---

### [파트 2] 숫자 신뢰 — 10분

**Q3. "이번 달 얼마 남았어요?" 라는 질문에 지금 바로 답할 수 있어요?**

듣고 싶은 것:
- 실시간으로 파악하는지
- 답하려면 뭘 봐야 하는지 (앱 열기, 계산, 모름)

→ 바로 안 된다면: "그걸 알려면 뭘 어떻게 해야 해요?"

---

**Q4. "오늘 얼마 써도 되는지" 숫자 하나가 딱 뜨면 그걸 믿을 것 같아요?**

듣고 싶은 것:
- 일단 반응이 긍정/부정 어느 쪽인지
- 믿으려면 뭐가 있어야 한다고 하는지

→ 믿겠다면: "어떤 조건이면 그 숫자를 못 믿게 될 것 같아요?"
→ 안 믿겠다면: "어떻게 계산됐는지 설명이 있어도요?"

---

**Q5. (핵심) 어떤 상황이면 그 숫자가 틀렸다고 느낄 것 같아요?**

이 질문이 가장 중요. 충분히 파고들 것.

예상 답변 유형과 후속 질문:
- "투자 이체가 지출로 잡히면" → "그런 적 실제로 있었어요?"
- "연간 보험료 같은 게 반영 안 되면" → "그런 비용 지금 어떻게 추적해요?"
- "salary가 아직 안 들어왔는데 잔액 기준이면" → "월급날이 밀린 경험이 있어요?"
- "내가 직접 확인 못 하면" → "확인한다는 게 구체적으로 어떤 걸 보고 싶다는 거예요?"

---

### [파트 3] 행동 패턴 — 8분

**Q6. CSV 업로드는 괜찮을 것 같아요?**

> 은행 앱에서 CSV 다운받아서 앱에 올리는 방식인데요.

듣고 싶은 것:
- 매월 할 의향이 있는지
- 이미 해본 적 있는지
- 귀찮음의 역치가 어디인지

→ 귀찮다면: "자동 연동이 없으면 아예 안 쓸 것 같아요?"
→ 괜찮다면: "한 달에 몇 번 정도 할 것 같아요?"

---

**Q7. Recurring bill 등록은 괜찮을 것 같아요?**

> 월세, 넷플릭스, 전기세 같은 걸 앱에 한 번 등록해두는 거예요.

듣고 싶은 것:
- 이 작업을 귀찮아하는지
- 지금도 어딘가에 이미 해두고 있는지 (달력, 노트)

→ 귀찮다면: "설정을 안 해도 되는 방식이 있다면요? (CSV에서 자동 감지)"

---

**Q8. 지금 쓰는 앱의 제일 짜증나는 점이 뭐예요?**

아무 도구나 OK. 은행 앱, 가계부 앱, Excel 모두 포함.

듣고 싶은 것:
- 가장 강하게 반응하는 불만 포인트
- 포기하게 된 경험이 있는지
- "이게 되면 쓸 텐데"라는 말이 나오는지

---

### [마무리] 5분

**Q9. 지금 제가 만들고 있는 걸 한 줄로 설명하면:**

> "투자 이체를 지출로 오해하지 않고, 다음 월급날까지 안전하게 쓸 수 있는 금액을 매일 알려주는 앱이에요. CSV 올리면 자동으로 분류해줘요."

이게 해결하고 싶은 문제예요?

→ 맞다면: "그럼 지금 당장 써보고 싶어요?"
→ 아니면: "실제 문제는 뭐예요?"

---

**Q10. 마지막으로 — 오늘 대화에서 제가 안 물어봤지만 중요한 게 있어요?**

```
직업/역할:
계좌 구성:
현재 도구:

[숫자 신뢰 신호]
- 믿겠다고 한 조건:
- 못 믿겠다고 한 조건:
- 가장 강하게 반응한 순간:

[불만 포인트 Top 3]
1.
2.
3.

[CSV/수동 입력 허용 의향]
- CSV: 매월 OK / 가끔 / 안 함
- Recurring 등록: OK / 귀찮음

[핵심 인용]
(직접 한 말 그대로 — 마케팅 카피 후보)

[safe-to-spend 정책 시사점]
- 더 보수적으로 해야 할 이유:
- 느슨하게 해도 될 이유:

[onboarding 시사점]
- 필수로 물어봐야 할 것:
- 안 물어봐도 될 것:
```

## 5. 3명 인터뷰 후 종합 판단 기준

| 질문 | 3명 중 2명 이상이면 → 액션 |
|------|--------------------------|
| "투자 이체 문제 실제로 겪었다" | safe-to-spend에서 intent 분리가 핵심임을 확인 |
| "숫자를 믿으려면 근거가 보여야 한다" | "Why this number?" 드릴다운을 MVP에서 절대 뺄 수 없음 |
| "연간 비용이 반영 안 되면 못 믿겠다" | sinking fund를 MVP에 포함 (지금은 V1.5였음) |
| "CSV 매월은 귀찮다" | import 빈도 목표를 월 1회가 아니라 분기 1회로 재설계 검토 |
| "recurring 등록이 귀찮다" | CSV 자동 감지 우선순위 올리기 |
